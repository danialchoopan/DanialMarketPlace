# مستندات REST API پروژه MarketPlaceCore

این سند شامل جزئیات تمام نقاط انتهایی (Endpoints) موجود در سیستم MarketPlaceCore است.

## احراز هویت (Authentication)

تمامی درخواست‌ها به جز ورود، نیازمند هدر احراز هویت هستند:
`Authorization: Bearer {token}`

### ۱. ورود به سیستم
- **آدرس:** `/api/Account/login`
- **متد:** `POST`
- **بدنه درخواست:**
```json
{
  "username": "admin",
  "password": "admin123"
}
```
- **پاسخ موفق (200 OK):**
```json
{
  "token": "JWT_TOKEN_HERE",
  "role": "Admin"
}
```

## محصولات (Products)

### ۱. دریافت لیست همه محصولات
- **آدرس:** `/api/Product`
- **متد:** `GET`
- **پاسخ موفق (200 OK):**
```json
[
  {
    "id": 1,
    "name": "گوشی موبایل آیفون 13",
    "price": 45000000,
    "stock": 10,
    "categoryId": 1
  }
]
```

### ۲. دریافت جزئیات محصول
- **آدرس:** `/api/Product/{id}`
- **متد:** `GET`

### ۳. ایجاد محصول جدید (فقط مدیر)
- **آدرس:** `/api/Product`
- **متد:** `POST`
- **دسترسی:** نقش Admin

## سفارشات (Orders)

### ۱. ثبت سفارش جدید
- **آدرس:** `/api/Order`
- **متد:** `POST`
- **بدنه درخواست:**
```json
{
  "items": [
    {
      "productId": 1,
      "quantity": 2
    }
  ]
}
```
- **توضیحات:** این عملیات به صورت تراکنشی انجام می‌شود. در صورت عدم موجودی کافی، کل فرآیند لغو (Rollback) می‌گردد.
- **پاسخ موفق (200 OK):**
```json
{
  "orderId": 10,
  "message": "سفارش با موفقیت ثبت شد"
}
```
- **خطا (400 Bad Request):** در صورت عدم موجودی یا خطای سیستمی.
